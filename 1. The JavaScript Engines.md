
Reading the documentation of node gave me the whole introduction to what is on the core.

I think we all know the node as a technology that gave us the opportunity to use the same language(JS) we used for the browser/client part(frontend) to also use for the backend environment.
So the difference between JS and Node.js is like, first one is the programming language for us to help control the browser environment(browser DOM), and the second is to control the system, the server.

OK, Node.js we need to write code with JS for backend environments, to create desktop and mobile apps.
BUT, there is one particular thing that made the Node shine.

In 2009 Firefox adds SpiderMonkey JavaScript compiler to its 3.5 version. Which is actually written in C and C++ that compiles our JS code, so it runs super fast.
From that time on, JS become not just a scripting language but best of two worlds: you write scripts, but they will be compiled in a browser as if it was a compiling language!

Node.js uses V8 engine which is considered best and which uses Chrome.

So on both **Node.js** server or on the **Chrome** browser, JavaScript is internally compiled by V8 with **just-in-time** (JIT) **compilation** to speed up the execution.

- Firefox has [**SpiderMonkey**](https://spidermonkey.dev/)
- Safari has [**JavaScriptCore**](https://developer.apple.com/documentation/javascriptcore) (also called Nitro)
- Edge was originally based on [**Chakra**](https://github.com/Microsoft/ChakraCore) but has more recently been [rebuilt using Chromium](https://support.microsoft.com/en-us/help/4501095/download-the-new-microsoft-edge-based-on-chromium) and the V8 engine.
